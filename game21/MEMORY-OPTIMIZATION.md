# Оптимизация памяти и управление игровыми сессиями 🧹

## Проблема

При создании игры создается сессия, которая может накапливать ресурсы в памяти:
- Таймеры в `turnTimers` Map
- Интервалы обновления времени
- Данные игры в Redis/Memory

## Решение

### 1. Жизненный цикл игровой сессии

```
Создание игры → Ожидание игроков → Игра → Завершение → Очистка
     ↓                ↓                ↓         ↓          ↓
  createGame()    joinGame()      playing    finished   cleanup
     1 час TTL                                          10 мин TTL
```

### 2. Автоматическая очистка

#### Redis/Memory TTL
- **При создании**: TTL = 3600 секунд (1 час)
- **После завершения**: Дополнительная очистка через 10 минут

#### Таймеры хода
- Очищаются при переходе хода
- Очищаются при завершении игры
- Очищаются при таймауте

#### Интервалы обновления
- Запускаются при старте игры
- Останавливаются при завершении игры
- Проверяют статус игры каждую секунду

### 3. Периодическая очистка

Каждые **5 минут** сервер:
1. Сканирует все игры
2. Находит завершенные игры старше 10 минут
3. Удаляет их из хранилища
4. Очищает связанные ресурсы (таймеры, интервалы)

```javascript
// Автоматически запускается в server.js
setInterval(async () => {
  // Очистка завершенных игр
}, 5 * 60 * 1000);
```

### 4. Мониторинг

#### API endpoint для статистики
```bash
GET /api/stats
```

Возвращает:
```json
{
  "success": true,
  "gameManager": {
    "activeTimers": 2,
    "timerGameIds": ["game_123", "game_456"]
  },
  "intervals": {
    "activeIntervals": 2,
    "intervalGameIds": ["game_123", "game_456"]
  }
}
```

### 5. Что происходит с ресурсами

| Ресурс | Создание | Очистка | Время жизни |
|--------|----------|---------|-------------|
| **Игровая сессия** | `createGame()` | Redis TTL | 1 час |
| **Таймер хода** | `startTurnTimer()` | `clearTurnTimer()` | 30 сек или до действия |
| **Интервал обновления** | `start_game` | `clearGameInterval()` | До завершения игры |
| **Завершенная игра** | `determineWinners()` | Периодическая очистка | 10 минут |


## Логи для мониторинга

Следите за этими сообщениями в консоли:

- ✅ `⏱️ Таймер запущен для игры` - таймер создан
- ✅ `⏹️ Таймер остановлен для игры` - таймер очищен
- ✅ `🧹 Игра завершена, ресурсы очищены` - игра завершена
- ✅ `🧹 Интервал очищен для игры` - интервал остановлен
- ✅ `🗑️ Удалена завершенная игра` - старая игра удалена
- ⚠️ Если таймеры/интервалы не очищаются - возможна утечка!
