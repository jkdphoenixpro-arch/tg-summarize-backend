# Таймер хода игрока - 30 секунд

## Описание
Добавлен таймер на 30 секунд для каждого хода игрока. Если игрок не делает ход в течение 30 секунд, автоматически засчитывается "пас" (stand).

## Изменения

### Серверная часть (game21/server/)

#### gameManager.js
- Добавлено свойство `TURN_TIMEOUT = 30000` (30 секунд)
- Добавлено хранилище таймеров `turnTimers` (Map)
- Добавлено поле `turnStartTime` в состояние игры для отслеживания начала хода
- Добавлены методы:
  - `startTurnTimer(gameId)` - запускает таймер для текущего игрока
  - `clearTurnTimer(gameId)` - останавливает таймер
  - `handleTurnTimeout(gameId)` - обрабатывает истечение времени (автоматический пас)
  - `getRemainingTime(game)` - возвращает оставшееся время в секундах

#### server.js
- Передача `io` в `GameManager` для отправки событий при таймауте
- Добавлена функция `emitGameUpdate()` для отправки обновлений с информацией о времени
- Добавлен интервал обновления времени каждую секунду во время игры
- Добавлена обработка события `turn_timeout` для уведомления клиентов

### Клиентская часть (game21/client/)

#### Game.jsx
- Добавлено состояние `remainingTime` для отображения оставшегося времени
- Добавлено состояние `showTimeoutMessage` для показа сообщения о таймауте
- Добавлен обработчик события `turn_timeout` от сервера
- Добавлен визуальный таймер с цветовой индикацией:
  - Зеленый: > 10 секунд
  - Оранжевый: 5-10 секунд
  - Красный: < 5 секунд
- Добавлено всплывающее сообщение при таймауте

#### Game.css
- Добавлены стили для таймера (`.turn-timer-container`, `.turn-timer`)
- Добавлены анимации:
  - `pulse-warning` - пульсация при < 10 секундах
  - `pulse-critical` - интенсивная пульсация при < 5 секундах
  - `fadeInOut` - появление/исчезновение сообщения о таймауте
- Адаптивные стили для разных размеров экрана

## Как это работает

1. **Начало игры**: При старте игры запускается таймер для первого игрока
2. **Ход игрока**: 
   - При действии "Hit" таймер сбрасывается и запускается заново
   - При действии "Stand" таймер останавливается и переходит к следующему игроку
3. **Истечение времени**: 
   - Через 30 секунд автоматически вызывается `stand` для текущего игрока
   - Отправляется событие `turn_timeout` всем клиентам
   - Показывается уведомление игроку
4. **Визуализация**: 
   - Таймер обновляется каждую секунду
   - Цвет меняется в зависимости от оставшегося времени
   - Анимация привлекает внимание когда времени мало

## Тестирование

Для тестирования:
1. Запустите игру с 2+ игроками
2. Дождитесь своего хода
3. Не делайте никаких действий 30 секунд
4. Должен автоматически произойти "пас"
5. Ход перейдет к следующему игроку

## Настройка

Чтобы изменить время хода, отредактируйте константу в `gameManager.js`:
```javascript
this.TURN_TIMEOUT = 30000; // Время в миллисекундах (30000 = 30 секунд)
```
